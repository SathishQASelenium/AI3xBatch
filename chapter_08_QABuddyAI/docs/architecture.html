02 Pipeline

Four stages, one collection. Ingest and Ask share the same BGE-M3 encoder so a query and a chunk land in the same vector space.

① Ingest

  1. per-source loader reads file
  2. normalize → chunk (per-source strategy)
  3. BGE-M3 encode — dense 1024d + lexical sparse, one pass
  4. upsert to Qdrant, stable uuid5 ids
  5. manifest diff — unchanged files skip
→

② Store

  1. one collection qabuddy_kb
  2. named vectors: dense + sparse
  3. payload indexes: source_type, repo, path, ticket_key, tc_id
  4. embedded locally · server mode on droplet
→

③ Ask

  1. condense follow-up against last 3 turns
  2. LLM query rewrite → 3 variants
  3. dense + sparse search, filtered by selected sources
  4. RRF fuse (k=60) → rerank top-12 → keep top-6
  5. score < 0.22 → "not in KB", no invented answer
  6. mode prompt + glossary → Groq gpt-oss-120b, streamed
→

④ UI

  1. Flask + SSE chat stream
  2. source filter chips, KB stats
  3. citation cards — file:line, VWO-123, doc p.X, build #N
  4. ingest panel (same process as server)

03 Why these pieces

BGE-M3

One open (MIT) model gives dense semantics and lexical sparse weights in a single pass — exact identifiers like VWO-123 or NoSuchElementException retrieve alongside fuzzy intent. 8k context, runs on CPU.

Qdrant

Named dense + sparse vectors on one point, payload filters, snapshots, small footprint (Rust). Embedded mode locally; a server container on the droplet — switched by one env var.

Cross-encoder rerank

The real answer-quality and token-efficiency lever. Only the top-6 chunks (~2.5–3k tokens) ever reach the LLM, keeping context tight and answers grounded.

gpt-oss-120b on Groq

Open-weight brain, hosted for speed. Swappable to a local Ollama instance by changing LLM_BASE_URL / LLM_MODEL only.

06 Trust chain

Every chunk payload carries enough to cite exactly, and to click through:

source_type path repo line_start / line_end ticket_key tc_id page build_id
Relevance gate — if the best rerank score falls below 0.22, QABuddy states the knowledge base doesn't contain the answer, rather than inventing one.
